    第12章 DOM2和DOM3

    DOM1主要定义的是HTML和XML文档的底层结构。DOM2和DOM3则在这个结构的基础上引入了更多的交互能力，也支持了更高级的XML特性。

    • DOM2级核心，在1级的核心技术上构建，为节点添加更多的方法和属性
    • DOM2级视图，为文档定义了基于样式信息的不同视图
    • DOM2级事件，说明了如何使用事件与DOM文档交互
    • DOM2级样式，定义了如何以变编程的方式来访问和改变CSS样式信息
    • DOM2级遍历和范围，引入了遍历DOM文档和选择其特定部分的新结构
    • DOM2级HTML，在1级的HTML基础上构建，添加了更多属性、方法和新接口

    DOM3级又增肌了“Xpath”模块和“加载与保存”模块

    # DOM变化

    针对XML命名空间的变化

    <html xmlns="http://www.w3.org/1999/xhtml">
	<head>
		<title>jaybai title</title>
	</head>
	<body>
		hello jaybai!
	</body>
</html>

    加前缀，避免一些冲突

    <xhtml:html xmlns="http://www.w3.org/1999/xhtml">
	<xhtml:head>
		<xhtml:title>jaybai title</xhtml:title>
	</xhtml:head>
	<xhtml:body>
		hello jaybai!
	</xhtml:body>
</xhtml:html>

    还可以混合使用多个命名空间

    <html xmlns="http://www.w3.org/1999/xhtml">
	<head>
		<title>jaybai title</title>
	</head>
	<body>
		<svg xmlns="http://www.w3.org/2000/svg" version="1.1" viewBox="0 0 100 100" style="width:100%;height:100%">
			<rect x="0" y="0" width="100" height="100" style="fill:red" />
		</svg>
	</body>
</html>

    Node类型的变化，DOM2中包含下列特定于明明空间的属性

    • localName：不带命名空间前缀的节点名称
    • namesapceURI：命名空见URI或者null
    • prefix：命名空间前缀或者null

    DOM3在此技术上又加了一些

    • isDefaultNamespace(namesapceURI)：在制定的namespaceURI是当前接待你的默认明明空间的情况下返回true
    • lookupNamespaceURI(prefix)：返回给定的prefix的命名空间
    • lookupPrefix(namespaceURI)：返回给定namespaceURI的前缀

    Document类型的变化,DOM2中加了下列东东：

    • createElementNS(namespaceURI, tagName)
    • createAttributeNS(namespaceURI, attributeName)
    • getElementsByTagNameNS(namespaceURI, tagName)

    Element类型的变化，DOM2中加了以下东东：

    • getAttributeNS(namespaceURI, localName)
    • getAttributeNodeNS(namespaceURI, localName)
    • getElementsByTagNameNS(namespaceURI, tagName)
    • hasAttributeNS(namespaceURI, localName)
    • removeAttributeNS(namespaceURI, localName)
    • setAttributeNS(namespaceURI, qualifiedName, value)
    • setAttributeNodeNS(attNode)

    NameNodeMap类型的变化，DOM2中加了以下东东：

    • getNameItemNS(namespaceURI, localName)
    • removeNameItemNS(namespaceURI, localName)
    • setNameItemNS(node)

    其他方面的变化

    DocmentType类型的变化,新增：

    • publicId
    • systemId
    • internalSubset，访问包含在文档类型声明的额外定义

    Docment类型的变化,新增：

    • importNode(要复制的节点， 是否复制子节点)
    • defaultView，指向拥有给定文档的窗口或者框架的指针（IE不支持，parentWindow代替）
    • docment.implementation.createDocumentType(文档类型名称， publicId， systemId)
    • docment.implementation.createDocument(namespaceURI，文档元素的标签名， 新文档的文档类型)
    • docment.implementation.createHTMLDocument(新建文档的标题)

    Node类型的变化,新增：

    • isSupported(特性名， 特性版本号)
    • isSameNode(node)，引用同一个对象返回true
    • isEqualNode(node)，2个节点属性值相等返回true
    • setUserData(要设置的值， 实际的数据， 处理函数)

    这里的最后的处理函数接受5个参数

    1. 操作类型的数值（1=复制，2=导入，3=删除，4=重命名）
    2. 数据键
    3. 数据值
    4. 源节点
    5. 目标节点

    框架的变化,新增：

    • contentDocument，指向表示框架内容的文档对象的指针

    # 样式

    访问元素的样式

    CSS样式中有"-"的要转化为驼峰才能通过JS访问

    例如常见的属性对应值：

    CSS属性 JavaScript属性
    background-image style.backgroundImage
    color style.color
    display style.display
    font-family style.fontFamily

    float转的话DOM2规定是cssFloat,IE用的是styleFloat

    DOM样式属性和方法

    • cssText，style中的CSS代码
    • length，元素CSS属性的数量
    • parentRule，CSSRule对象
    • getPropertyCSSValue(propertyName)，返回给定属性值的CSSRUle类型
    • getPropertyPriority(propertyName)，给定属性使用了!important，返回“important”，否则返回空字符串
    • getPropertyValue(propertyName)，返回给定属性的字符串
    • item(index)，返回给定位置的CSS属性的名称
    • removeProperty(propertyName)，从样式中删除给定属性
    • setProperty(propertyName, value, priority)，将给定属性设置为相应的值，并加上优先权标志

    计算的样式

    DOM2增强了document.defaultView提供了以下方法

    • getComputedStyle(要取得计算样式的元素，伪元素字符串)

    操作样式表

    CSSStyleSheet继承自StyleSheet，从它那继承如下：

    • disabled
    • href
    • meadia
    • ownerNode
    • parentStyleSheet
    • title
    • type
    • cssRules
    • ownerRule
    • deleteRule(index)
    • inserRule(rule, index)

    CSSRule对象表示样式表中每一条规则，包含以下：

    • cssText
    • parentRule
    • parentStyleSheet
    • selectorText
    • style
    • type
    //最多可以加入4095条规则
function insertRule(sheet, selectorText, cssText, position){
	if(sheet.insertRule){
		sheet.insertRule(selectorText + "(" + cssText + ")", position);
	}else if(sheet.addRule){ //IE
		sheet.addRule(selectorText, cssText, position);
	}
}

insertRule(document.styleSheet[0], "body", "background-color: silver", 0);
function deleteRule(sheet, index){
	if(sheet.deleteRule){
		sheet.deleteRule(index);
	}else if(sheet.removeRule){ //IE
		sheet.removeRule(index);
	}
}

deleteRule(document.styleSheet[0], 0);

    元素大小

    偏移量(外部距离)

    • offsetHeight
    • offsetWidth
    • offsetLeft
    • offsetTop
    • offsetParent，包含元素的引用

    客户区大小（内部大小）

    • clientHeight
    • clientWidth

    滚动大小

    • scrollHeight
    • scrollWidth
    • scrollLeft
    • scrollTop

    确定元素大小

    getBoundingClientRect()方法返回一个矩形对象，包含

    • left
    • top
    • right
    • bottom

    # 遍历

    DOM2定义了2个辅助遍历的DOM结构的类型

    1. NodeIterator
    2. TreeWalker

    DOM遍历是深度优先遍历

    document.createNodeIterator()方法创建实例，需要4个参数：

    1. root，搜索起点节点
    2. whatToShow，要访问哪些节点的数字代码
    3. filter，NodeFilter对象或者表示应该接受还是拒绝某种特定节点的函数
    4. entityReferenceExpansion，是否要拓展实体引用，在HTML页面中没用

    whatToShow参数是一个位掩码，定义如下：

    • NodeFilter.SHOW_ALL：显示所有类型的节点
    • NodeFilter.SHOW_ELEMENT：显示元素节点
    • NodeFilter.SHOW_ATTRIBUTE：显示特性节点，由于DOM结构原因，不能用这个值
    • NodeFilter.SHOW_TEXT：显示文本节点
    • NodeFilter.SHOW_CDATA_SECTION：显示CDATA节点，HTML无效
    • NodeFilter.SHOW_ENTITY_REFERENCE：显示实体引用节点，HTML无效
    • NodeFilter.SHOW_ENTITYE：显示实体节点，HTML无效
    • NodeFilter.SHOW_PROCESSING_INSTRUCTION：显示处理指令节点，HTML无效
    • NodeFilter.SHOW_COMMENT：显示注释节点
    • NodeFilter.SHOW_DOCUMENT：显示文档节点
    • NodeFilter.SHOW_DOCUMENT_TYPE：显示类型的节点
    • NodeFilter.SHOW_DOCUMENT_FRAGMENT：显示文档片段节点，HTML无效
    • NodeFilter.SHOW_NOTAION：显示符号节点

    TreeWalker

    TreeWalker是NodeIterator的高级版本。除了nextNode()和previousNode()相同外，还提供了其他方法：

    • parentNode()
    • firstChild()
    • lastChild()
    • nextSibling()
    • previousSibling()

    创建方法和参数跟NodeIterator类似。

    # 范围

    DOM中的范围

    Range实例包含以下属性

    • startContainer，包含范围起点的节点（选取中第一个节点的父节点）
    • startOffset，范围在startContainer中起点的偏移量
    • endContainer，包含范围终点的节点（选取中最后一个节点的父节点）
    • endOffset，范围在endContainer中起点的偏移量
    • commonAncestorContainer，startContainer和endContainer的共同祖先节点在文档书中位置最深的那个

    用DOM范围实现简单选择

    • selectNode(node)，选择整个节点，包含子节点
    • selectNodeContents(node)，只选择节点的子节点

    为了精细控制，还可以使用一下方法

    • setStartBefore(refNode)
    • setStartAfter(refNode)
    • setEndBefore(refNode)
    • setEndAfter(refNode)

    用DOM范围实现负责选择

    • setStart(参照节点， 偏移量)
    • setEnd(参照节点， 偏移量)

    操作DOM范围中的内容

    • deleteContents()
    • extractContents()，区别于上面的删除，会返回删除的值
    • appendChild()

    插入DOM返回中的内容

    • insertNode()
    • surroundContents()

    折叠DOM范围

    • collapse(boolean)， true表示折叠到起点，false表示折叠到终点

    比较DOM范围

    • compareBoundaryPoints(比较方式的常量，比较的范围)

    比较方式的常量有：

    • Range.START_TO_START(0)，比较第一个范围和第二个范围的起点，以下的类似
    • Range.START_TO_END(1)
    • Range.END_TO_END(2)
    • Range.END_TO_START(3)

    compareBoundaryPoints()返回值：

    • -1，第一个范围中的点位于第二个范围中的点之前
    • 0，相等
    • 1，第一个范围中的点位于第二个范围中的点之后

    复制DOM范围

    var newRage = range.cloneRange();

    清理DOM范围

    • detach()方法
