第17章 错误处理与调试

JS中的try-catch-finally语法和Java中的没有什么区别。需要注意的是catch出来的error对象不同浏览器的对象有点不一致，一般message都有。

fuction a(){
	try{
		//可能导致错误的代码
	}catch(error){
		alert(error.message);
	}finally{
		//收尾工作
	}
}

7种错误类型：

错误类型 说明 举例
Error 基类型，其他error都继承自Error
EvalError 没有把eval()当做函数调用就会报这个错误。（也有的浏览器实现不一样） new eval();
eval = foo;
RangeError 数值超出范围的时候报这个错误 var i = new Array(-20);
ReferenceError 找不到对象的时候 var a = b;//b未定义
SyntaxError 错误无法的字符串传入eval()中会报这个错误 eval("a ++ b");
TypeError 执行特定类型的操作时，变量不符合要求，常出现在不检查就直接用参数的情况下 var a = new 10;
URIError 使用encodeURI()或decodeURI()时URI格式不正确的时候 
//在catch中检查不同的错误，这里跟java无限catch的语法不一样，需要自己if-else
fuction a(){
	try{
		//可能导致错误的代码
	}catch(error){
		if（error instanceof TypeError）{
			//处理TypeError错误
		}else if（error instanceof ReferenceError）{
			//处理ReferenceError错误
		}else{
			//处理其他错误
		}
	}finally{
		//收尾工作
	}
}

抛出错误也是用throw，但是可以随意抛都不报错，一般自定义错误这样操作：

fuction JaybaiError(message){
	this.name = "JaybaiError";
	this.message = message;
}

JaybaiError.prototype = new Error();

throw new JaybaiError("this is a Error");

常见的错误类型：

  1. 类型转换错误
  2. 数据类型错误
  3. 通信错误

非致命错误：

  1. 不影响用户的主要任务
  2. 只影响页面的一部分
  3. 可以恢复
  4. 重复相同的操作可以消除错误

致命错误

  1. 应用程序根本无法继续运行
  2. 错误明显的影响到了用户的主要操作
  3. 会导致其他连带错误

常见的IE错误：

  1. 操作终止：IE8之前如果未加载完页面又操作页面就会报这个
  2. 无效字符：特殊字符，特别是word中粘贴过来类似减号的字符
  3. 未找到成员：导致错误的一定是COM对象，对象被销毁后又给对象赋值就会报这个
  4. 未知运行时错误：使用innerHTML或者outerHTML把块级元素插入行内元素或者访问表格任意部分
  5. 语法错误：特别要注意服务端回写到html中的内容
  6. 系统无法找到指定资源：url不符合规范js请求不才能超过2083，浏览器自身限制2048

如何避免浏览器响应JS错误:

  1. 在可能发生错误的地方使用try-catch
  2. 使用window.onerror兜底
