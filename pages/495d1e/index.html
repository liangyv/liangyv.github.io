    第8章 BOM

    ES是JS的核心，但是如果再Web中的话，BOM无疑才是核心。因为BOM都是浏览器的提供的，所以各自为政，于是浏览器之间共有的对象就成了事实上的标准。然后W3C把最后将BOM的主要方法纳入了H5规范。

    # window对象

    全局作用域

    var name = "jaybai";
function sayName( ){
	alert(this.name);
}

alert(windos.name); //jaybai
sayName(); //jaybai
window.sayName(); //jaybai

delete window.name; //IE9之前报错，其他返回false
window.age = 18;
delete window.age; //IE9前报错，其他返回true
alert(windos.name); //jaybai
alert(windos.age); //undefined

var a = b; //报错，并未定义
var c = window.d; //c的值是undefind

    全局JS对象实际上都是window对象的属性

    Windows Mobile平台下的IE不允许直接给window对象的属性赋值。但是可以这么调用。

    窗口关系及框架

    <html>
	<head>
		<title>框架例子</title>
	</head>
	<frameset rows="160,*">
		<frame src="a.html" name="a">
		<frameset cols="50%,50%">
			<frame src="b.html" name="b">
			<frame src="c.html" name="c">
		</frameset>
	</frameset>
</html>

    可以通过window.frames[0]或者window.frames["a"]来引用上方框架，不过最好使用top.window.frames[0]这样的方式来引用。

    • top始终事项最高（最外）层框架，也就是浏览器窗口。
    • parent始终指向当前框架的直接上层框架，某些时候parent等于top，没有框架的时候parent一定等于top等于window。
    • 除非最高层窗口是通过window.open()打开的，否则其window对象的name属性不会包含任何值
    • self始终指向window
    • 以上这些属性都是window的对象属性

    使用框架的情况下，浏览器会有多个Global对象。每个框架都有自己的构造函数，一一对应但是并不相等。例如：top.Object不等于top.frames[0].Object。

    导航和打开窗口

    window.open(需要加载的URL, 窗口目标, 特性字符串, 新页面是否取代浏览器历史记录总当前加载页的布尔值);

    第3个参数有如下取值，逗号分隔，可以多个：

    设置 值 说明
    fullscreen yes或no 浏览器窗口是否最大化，仅限IE
    height 数值 新窗口高度，不能小于100
    left 数值 新窗口的左坐标，不能是负值
    location yes或no 是否在浏览器中显示地址栏，设成no根据不同浏览器可能会隐藏或者禁用
    menubar yes或no 是否在浏览器窗口显示菜单栏，默认no
    resizeable yes或no 是否可以通过拖动浏览器窗口的边框改变其大小，默认no
    scrollbars yes或no 如果内容在视口中显示不下，是否可以滚动，默认no
    status yes或no 是否在浏览器窗口中显示状态栏，默认no
    toolbar yes或no 是否在浏览窗口中显示工具栏，默认no
    top 数值 新窗口的上坐标，不能是负值
    width 数值 新窗口的宽度，不能小于100
    • 对于window.open()的窗口，如果有赋值给实例，可以通过close()方法关闭，主窗口不行。
    • 对于window.open()的窗口，如果设置opener = null;可以告诉浏览器新创建的标签页不需要与打开他的标签页通信，一旦设置不可恢复。
    var blocked = false;

try{
	var newWin = window.open("https://www.juzilicai.com", "_blank");
	if(newWin == null){ //一般是浏览器内置屏蔽程序
		blocked = true;
	}
}catch(ex){ //一般是浏览器拓展或者插件
	blocked = true;
}

if(blocked){
	alert("大哥，你的弹出窗口被人拦截了！");
}

    间歇调用和超时调用

    //超时调用
//不建议这么直接传递字符串的用法，但是可以这么写
setTimeout("alert('hello jaybai!')", 1000);
	
//推荐方式
setTimeout(function( ){
	alert("hello jaybai!");
}, 1000); //延迟1秒

var tid = setTimeout(function( ){
	alert("hello baijie!");
}, 2000);

//在制定超时时间前调用还可以取消
clearTimeout(tid);

    JS是单线程语言，延迟时间只是表示在这个时间后，会把当前任务添加到一个执行队列中去。如果队列是空的那就立刻执行，如果队列不为空，还要等待队列前面的代码执行完后再执行。

    //间歇调用
//不建议这么直接传递字符串的用法，但是可以这么写
setInterval("alert('hello jaybai!')", 1000);
	
//推荐方式
setInterval(function( ){
	alert("hello jaybai!");
}, 1000); //延迟1秒

var iid = setInterval(function( ){
	alert("hello baijie!");
}, 2000);

//调用还可以取消
clearTimeout(iid);

    不建议使用间歇调用，因为后一个间歇调用可能会在前一个间歇调用结束之前启动。可以用超时调用来模拟间歇调用。

    # location对象

    提供了当前窗口中加载的文档有关的信息，还提供了一些导航功能。既是window对象的属性，也是document对象的属性（window.location和document.location引用同一个对象）。

    location对象的所有属性:

    属性名 例子 说明
    hash "#aaa" URL中的hash(#号后面的)，没有的话返回空串
    host "www.juzilicai.com (opens new window):443" 返回服务器名称和端口号（如果有）
    hostname "www.juzilicai.com (opens new window)" 返回不带端口号的服务器名称
    href "https://www.juzilicai.com (opens new window)" 返回当前完整URL
    pathname "/about/" 返回URL中的目录或者文件名
    port "443" 返回URL中指定的端口号，不存在返回空
    protocol "https:" 返回页面使用的协议
    search "?channelId=4" 返回URL的查询字符串（?后面的）

    查询字符串参数

    //直接访问不方便，需要拆分这个查询串
function getQueryStringArgs( ){
	//取得查询字符串并去掉开头的？
	var qs = (location.search.length > 0 ? location.search.substring(1) : ""),
	//报错数据的对象
	args = {},
	//取得每一项
	items = qs.length ? qs.split("&") : [],
	item = null,
	name = null,
	value = null,
	i = 0;
	len = items.length;
	
	//逐个解析参数加到args中去
	for(i=0; i < len; i++){
		item = items[i],split("=");
		name = decodeURIComponent(item[0]);
		value = decodeURIComponent(item[1]);
		
		if(name.length){
			args[name] = value;
		}
	}
	
	return args;
}

    检测插件

    因为IE和非IE检测方式不一致，一般不通过通用的方案来解决。需要单独对每种插件进行浏览器判断

    //检测插件（非IE）
function hasPlugin(name){
	name = name.toLowerCase();
	for(var i=0; i < navigator.plugins.length; i++){
		if(navigator.plugin[i].name.toLowerCase().indexOf(name) > -1){
			return true;
		}
	}
	return false;
}

//检测IE中的插件
function hasIEPlugin(name){
	try{
		new ActiveXObject(name);
		return true;
	}catch(ex){
		return false;
	}
}

//检查Flash插件
function hasFlash( ){
	var r = hasPlugin("Flash");
	if(!r){
		r = hasIEPlugin("ShockwaveFlash.ShockwaveFlash");
	}
	return r;
}

//检测QuickTime插件
function hasQucikTime( ){
	var r = hasPlugin("QuickTime");
	if(!r){
		r = hasIEPlugin("QuickTime.QuickTime");
	}
	return r;
}

alert(hasFlash());
alert(hasQucikTime());

    plugins集合有一个方法refresh()用来刷新plugins。接受一个布尔型参数，true的话会重新加载包含插件的所有页面，否则只更新plugins集合。
