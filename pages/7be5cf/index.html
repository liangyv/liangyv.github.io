    第10章 DOM

    DOM(文档对象模型)是针对HTML和XML文档的一个API。描绘了一个层次化的节点树，脱胎于DHTML。

    IE中的DOM对象都是以COM对象的形式实现的。意味着IE中的DOM对象与原生JavaScript对象的行为或者特点并一定一致。

    # 节点层次

    HTML页面都是

    包围的，xml中无限制。这个不多写了，大家都理解。

    # Node类型

    DOM1级定义了一个Node接口，该接口将有DOM中的所有节点类型实现。除IE外其他都可以访问这个类型。每个节点都有nodeType属性，节点类型有12个数值常量表示，任何节点都必居其一：

    1. Node.ELEMENT_NODE
    2. Node.ATTRIBUTE_NODE
    3. Node.TEXT_NODE
    4. Node.CDATA_SECTION_NODE
    5. Node.ENTITY_REFERENCE_NODE
    6. Node.ENTITY_NODE
    7. Node.PROCESSING_INSTRUCTION_NODE
    8. Node.COMMENT_NODE
    9. Node.DOCUMENT_NODE
    10. Node.DOCUMENT_TYPE_NODE
    11. Node.DOCUMENT__FRAGMENT_NODE
    12. Node.NOTATION_NODE

    节点的具体信息

    • nodeName
    • nodeValue

    节点关系

    var a = someNode.childNodes[0]; //第1个节点
var b = someNode.childNodes.item(1); //第2个节点
var c = someNode.childNodes.length; //访问的这一刻节点数量
//节点转数组
function convertToArray(nodes){
	var array = null;
	try{
		array = Array.prototype.slice.call(nodes, 0); //非IE浏览器有效
	}catch(ex){
		array = new Array();
		for(var i=0, len=nodes.length; i < len; i++){ //IE只能自己循环
			array.push(nodes[i]);
		}
	}
	return array;
}
//如果在父节点下只有一个子节点，那它的前后节点都是null
if(someNode.nextSibling === null){
	alert("在父节点下这是最后一个节点了。");
}else if(someNode.previousSibling === null){
	alert("在父节点下这是第一个节点。");
}

//所以获得第一个和最后一个节点靠谱的办法还是用
var a = someNode.firstChild; 
var b = someNode.lastChild;

if(someNode.hasChildNodes()){
	alert("有子节点");
}

    所有节点都还有一个ownerDocument属性，直接指向文档节点，不需要层层回溯，直接访问顶层。

    操作节点

    • 新增节点，appendChild(新节点)、insertBefore(新节点， 插入哪个节点前);
    • 替换节点，replaceChild(新节点， 旧节点);
    • 删除节点，remove(待删除的节点);
    • 克隆节点，cloneNode(是否深度克隆);就算深度也是不会复制JS属性的，只复制特性、子节点

    替换和删除节点后被移除掉的节点都还在文档，只是没有位置了。克隆也一样，克隆出来的也是孤魂野鬼，需要插入到文档结构中去才有意义。

    Document类型

    JS中Document类型表示文档，在浏览器中document对象是HTMLDocment(继承自Document类型)的一个实例，表示整个HTML页面，document对象是window对象的一个属性。Document节点具有下列特征：

    • nodeType的值为9
    • nodeName的值为"#document"
    • nodeValue的值为null
    • parentNode的值为null
    • ownerDocument的值为null
    • 其子节点可能是一个DocumentType(最多一个)、Element(最多一个)、ProcessingInstruction或Comment。

    2个快捷方式访问Document的子节点

    1. documentElement属性始终指向HTML页面中的元素
    2. childNodes列表

    其他属性：

    • document.body属性，直接指向元素。
    • document.doctype属性，可以访问<!DOCTYPE>，但是浏览器支持差别很大，不推荐了。
    • document.title，指向元素
    • document.URL，包含页面完整的URL
    • document.domain，只包含页面的域名（可设置，但是要跟URL同步，可以解决部分跨域问题）
    • document.referrer，来源页面，如果没有可能是空串

    其他方法：

    • getElementById()，按id查，严格匹配大小写，IE8以下不区分，如果多个返回第一个
    • getElementByTagName()，按标签名返回，是0到多个的NodeList，在HTML中返回HTMLCollection对象。甚至变态到可以支持传入"*"，搜索所有的元素。HTML中不区分大小写，标准XML是区分的。
    • getElementByName()，返回给定name特征的所有元素，单选按钮这种除外

    特殊集合：

    • document.anchors，包含文档中所有带name特性的元素
    • document.applets，包含文档中所有的元素，不建议使用了
    • document.forms，包含文档中所有的
      元素
    • document.images，包含文档中所有的元素
    • document.links，包含文档中所有带href特性的元素

    DOM一致性检测

    DOM分多级也包含多个部分，可以通过一下方式检测

    //虽然这样可以检测，但是最好还是做能力检测
var hasXmlDom = document.implementation.hasFeature("XML", "1.0");
    功能 版本号 说明
    Core 1.0、2.0、3.0 基本的DOM，描述表现文档的节点树
    XML 1.0、2.0、3.0 Core的XML扩展，添加了对CDATA、处理指令及实体的支持
    HTML 1.0、2.0 XML的HTML扩展，添加了丢HTML特有元素及实体的支持
    Views 2.0 基于某些样式完成文档的格式化
    StyleSheets 2.0 将样式表关联文档
    CSS 2.0 对层叠样式表1级的支持
    CSS2 2.0 对层叠样式表2级的支持
    Events 2.0、3.0 常规的DOM事件
    UIEvents 2.0、3.0 用户界面事件
    MouseEvents 2.0、3.0 由鼠标引发的事件
    MutationEvents 2.0、3.0 DOM树变化时引发的事件
    HTMLEvents 2.0 HTML4.01事件
    Range 2.0 用于操作DOM树种某个范围的对象和方法
    Traversal 2.0 遍历DOM树的方法
    LS 3.0 文件与DOM树之间的同步加载和保存
    LS-Async 3.0 文件与DOM树之间的异步加载和保存
    Validation 3.0 在确保有效的前提下修改DOM树的方法

    文档写入祖传4方法：

    • write(字符串)，原样写入
    • writeln(字符串)，字符串加上换行符\n
    • open()，打开页面输出流（页面加载期间不用调用这玩意）
    • close()，关闭页面输出流（页面加载期间不用调用这玩意）

    Element类型

    Element类型用于表现XML或者HTML元素，提供了对元素的标签吗、子节点及特性的访问。具有以下特征：

    • nodeType的值为1
    • nodeName的值为元素的标签名（tagName也是一样，注意大小写，HTML都是大写,XML中一般是一致的）
    • nodeValue的值为null
    • parentNode可能是Document或Element
    • 其子节点可能是Element、Text、Comment、ProcessingInstruction、CDATASection或EntiyReference

    HTML标准特性（都有的）：

    • id
    • title
    • lang
    • dir，语言发现ltr（从左至右）rtl（从右至左）
    • className，与元素的class特性对应

    其他所有的HTML元素都有对应的可以访问的类型，具体就不写了，用的时候查一下就好了。

    操作特性的DOM方法主要有三个：

    • getAttribute(属性名);
    • setAttrubute(属性名， 值);
    • removeAttribute(属性名);

    有2类特殊的特性通过getAttribute()返回的值与属性值并不相同：

    1. style，通过getAttribute()返回包含CSS文本，通过对象属性访问返回一个对象
    2. 事件处理程序（onclick这种），通过getAttribute()返回的是代码的字符串，访问属性时返回一个JS函数或者null（未定义）

    attributes属性包含一个NamedNodeMap，NamedNodeMap对象有下列方法：

    • getNamedItem(name)
    • removeNamedItem(name)
    • setNamedItem(node)
    • item(pos)

    document.creatElement(完整元素标签)方法可以创建一个新元素。

    Text类型

    文本节点，包含的纯文本内容（可以是转移后的html）

    • nodeType的值为3
    • nodeName的值为"#text"
    • nodeValue的值为节点所包含的文本（data属性也一样）
    • parentNode是一个Element
    • 不支持（没有）子节点

    下列方法可以操作节点中的文本：

    • appendData(text)
    • deleteData(offset, count)
    • insertData(offset, text)
    • replaceData(offset, count, text)
    • splitText(offset)：从指定位置将当前文本节点分成2个文本节点
    • substringData(offset, count)
    • length

    document.creatTextNode()方法可以创建一个新文本节点。

    var a = document.creatElement("div");
a.className = "aaa";

var b = document.creatTextNode("hello ");
a.appendChild(b);

var c = document.creatTextNode("jayba");
a.appendChild(c);

alert(a.childNodes.length); //2

a.normalize(); //合并相邻的文本节点
alert(a.childNodes.length); //1
alert(a.firstChild.nodeValue); //"hello jaybai"

    浏览器在解析文档时用户不会创建相邻的文本节点。这种需要合并的情况只会在DOM操作中出现

    Comment类型

    注释类型。继承自text

    • nodeType的值为8
    • nodeName的值为"#comment"
    • nodeValue的值为注释的内容
    • parentNode是一个DocumentElement
    • 不支持（没有）子节点

    各种操作和属性跟text一样，除了没有splitText()方法

    CDATASection类型

    只针对XML文档，表示的是CDATA区域

    • nodeType的值为4
    • nodeName的值为"#cdata-section"
    • nodeValue的值为CDATA区域中的内容
    • parentNode是一个Document或Element
    • 不支持（没有）子节点

    各种操作和属性跟text一样，除了没有splitText()方法

    DocumentType类型

    在web浏览器中不常用，包含于文档的doctype相关的所有信息。

    • nodeType的值为10
    • nodeName的值为doctype的名称
    • nodeValue的值为null
    • parentNode是一个Document
    • 不支持（没有）子节点

    DOM1级中DocumentType对象不能动态创建，只能通过解析文档代码的方式创建。有3个属性

    • name
    • entities
    • notations

    DocumentFragment类型

    在所有节点中，只有DocumentFragment在文档中没有对应的标记。是一种“轻量级”的文档：

    • nodeType的值为11
    • nodeName的值为"document-fragment"
    • nodeValue的值为null
    • parentNode的值为null
    • 其子节点可能是Element、Text、Comment、ProcessingInstruction、CDATASection或EntiyReference

    document.creatDocumentFragment()方法可以创建一个新的文档片段。

    Attr类型

    元素特性在DOM中以Attr类型表示，可以访问Attr类型的构造函数和原型。

    • nodeType的值为11
    • nodeName的值是特性的名称
    • nodeValue的值为特性的值
    • parentNode的值为null
    • HTML中不支持（没有）子节点
    • XML中其子节点可能是Text或EntiyReference

    Attr对象有3个属性

    • name
    • value
    • specified，布尔值，表示是代码制定的还是默认的

    不建议直接访问特性节点。
