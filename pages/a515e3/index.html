第21章 Ajax和Comet

Ajax通信的主要对象是XMLHttpRequest，简称XHR

XHR接受3个参数：

  1. 请求的类型（get/post）
  2. 请求的URL
  3. 是否异步的boolean值

XHR先open()，再send();

响应包括：

  • responseText
  • responseXML
  • status
  • statusText

XHR对象的readyState属性如下，每次readyState变化都会触发一次onreadychange()方法：

取值 状态 含义
0 未初始化 尚未调用open()方法
1 启动 已调用open()方法，尚未调用send()方法
2 发送 已调用send()方法，尚未收到响应
3 接受 已收到部分响应
4 完成 已收到全部响应，而且可以在客户端使用了

默认情况下XHR请求会带上如下响应头：

  • Accept
  • Accept-Charset
  • Accept-Encoding
  • Accept-Language
  • Connection
  • Cookie
  • Host
  • Referer
  • User-Agent

FormData对象可以不指定头部，自动添加合适的。

XHR超时设置可以直接设置timeout属性，触发超时会调用ontimeout()方法

overrideMimeType()方法可以重写MIME类型。

XHR还支持6个进度事件：

取值 含义
loadstart 在接收到响应数据的第一个字节时触发
progess 在接收到响应期间不断触发
error 在请求发生错误时触发
abort 在因为调用abort()而终止连接时触发
load 在接收到完整响应数据时触发
loadend 在通信完成或者触发error、abort、load事件后触发

跨资源共享

Ajax受限于同源策略。可以通过CORS（Corss-Origin Resoure Sharing，跨资源共享）解决。主要就是在请求头部增加一个Origin头部，包含了请求页面的源信息。然后服务器上做修改，如果可以接受就在Access-Control-Allow-Origin头部中回发相同的源信息（如果公共资源可以发“*”）。
