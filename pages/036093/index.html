    第11章 DOM拓展

    DOM拓展有很多都是专有的，慢慢纳入规范。主要的扩展有2个：

    1. Selectors API（选择符API）
    2. HTML5

    # 选择符API

    根据CSS选择符选择与某个模式匹配的DOM元素。

    jQuery的核心就是通过CSS选择符查询DOM文档取得元素的引用。

    Selectors API Level1的核心是2个方法：

    1. querySelector()
    2. querySelectorAll()

    querySelector()方法

    //取得body元素
var body = document.querySelector("body");
//取得ID为a的元素
var body = document.querySelector("#a");
//取得类为b的第一个元素
var body = document.querySelector(".b");
//取得类为c的第一个图像元素
var body = document.querySelector("img.c");

    通过Document调用querySelector()查询整个文档，如果通过具体的Element调用querySelector()则回回在该元素后代元素的范围内查询匹配

    querySelectorAll()方法

    跟querySelector()类似，只是返回的是一个NodeList对象。

    matchesSelector()方法

    接受一个参数CSS选择符，如果调用元素与该选择符匹配返回true,否则返回false。

    老的浏览器支持不是特别完善，注意做兼容性处理

    # 元素遍历

    因为元素建的空格是不是返回文本接口浏览器有不同。所以Element Traversal API 为DOM元素添加了一下5个属性：

    1. childElementCount
    2. firstElementChild
    3. lastElementChild
    4. previousElementSibling
    5. nextElementSibling

    # HTML5

    HTML5规范围绕如何使用新增标记定义了大量的JavaScript API。和DOM有一些重叠。

    与类相关的扩充

    1. getElementsByClassName()方法，参数是1个或多个类名字符串，返回NodeList
    2. classList属性，用来应对多个类名操作其中部分的场景，集合类型是DOMTokenList

    DOMTokenList有length属性，可以用item()方法遍历，还有如果方法：

    • add(value)
    • contains(value)，是否存在给定值，返回true/false
    • remove(value)
    • toggle(value)，如果列表中已经存在给定值，删除它，如果没有，添加它。（开关方法）

    焦点管理

    var btn = document.getElementById("aButton");
b.focus();

alert(document.activeElement === b); //true
alert(document.hasFocus()); //true

    默认情况下，文档刚刚加载完document.activeElement保存的是document.body元素。加载期间document.activeElement的值为null

    HTMLDocument的变化

    HTML5对HTMLDocument进行了拓展:

    1. readyState属性。loading=正在加载，complete=已经加载完毕
    2. compatMode属性。CSS1Compat=标准模式，BackCompat=混杂模式
    3. head属性。

    字符集属性

    1. charset属性。默认值是"UTF-16"
    2. defaultCharset属性。浏览器及操作系统设置的默认字符集

    自定义数据属性

    //自定义数据属性以“data-”开头
<div id="a" data-a="123"></div>
var div = document.getElementById("a");
var a = div.dataset.a; //取自定义属性

    插入标记

    1. innerHTML属性。读出来就是HTML代码，写入的时候会转成HTML代码，但是各种浏览可能有细微差别
    2. outerHTML属性。会包含当前元素的代码。其他跟innerHTML类似
    3. insertAdacentHTML(pos, htmlCode)。

    insertAdacentHTML的pos只有以下取值：

    • beforebegin。当前元素之前插入一个紧邻的同级元素
    • afterbegin。当前元素之下插入一个子元素，排在子元素的第一个
    • beforeend。当前元素之下插入一个子元素，排在子元素组后一个
    • afterend。当前元素之后插入一个同级元素

    因为内存和性能问题，在插入元素之前，最好手工杉树要被替换的元素的所有事件处理程序和JS对象属性。

    循环插入的话建议先构建html字符串一次性调用插入方法，而不是每次循环去插入

    scrollIntoView()方法

    参数取值：

    • true/不传参数。让调用元素的顶部尽可能与视口顶部平齐
    • false。让调用元素尽可能全部出现，可能话底部与视口平齐

    # 专有扩展

    文档模式

    IE8引入了一个“文档模式”(document mode)的概念。决定了页面可以使用什么功能，决定了使用哪个级别的CSS，可以用那些JS API以及如果对待doctype

    chlidren属性

    IE9之前的版本因为处理文本节点中的空白符有差异，所以有了这个属性，只包含元素的子节点。

    contains()方法

    IE引入了这个方法方便查找某个节点是不是调用节点的后代节点。

    DOM Level3 compareDocumentPosition()也可以确认关系。返回一个掩码

    • 1，无关
    • 2，居前
    • 4，居后
    • 8，包含
    • 16，被包含

    插入文本

    跟innerHTML和outerHTML类似，但是没有纳入规范，所以个人不建议用了吧

    • innerText
    • outerText

    滚动

    也是没有被纳入H5规范的一些方法，一样最好没事别用了

    • scrollIntoViewIfNeeded(alignCenter)
    • scrollByLines(lineCount)
    • scrollByPages(pageCount)
