    第23章 离线检查

    # 离线检查

    常规方案是先通过navigator.online属性创建一个初始值，再根据监控window对象的online和offline事件来修改在线状态属性。

    # 应用缓存

    H5规定浏览器专门开辟一段缓存区域用来缓存文件。要想使用这个缓存，可以使用一个描述文件，这个文件的MIME类型必须是text/cache-manifest，内容类似于:

    CACHE MANIFEST
#Coment

file.js
file.css

    然后再html中指定该文件

    <html manifest="/offline.appcahe">

    应用缓存又一个API，核心对象是applicationCache对象，它有一个status属性，取值如下：

    取值 状态 描述
    0 无缓存 没有与页面相关的缓存
    1 闲置 应用缓存未得到更新
    2 检查中 正在下载描述文件并检查更新
    3 下载中 应用缓存正在下载描述文件中的指定资源
    4 更新完成 应用缓存已经更新了资源，而且所有资源都已下载完毕，可以通过swapCache()来使用了
    5 废弃 应用缓存的描述文件已经不存在了，因此无法再访问应用缓存

    应用缓存还有很多相关的事件：

    • checking
    • error
    • noupdate
    • downloading
    • progress
    • updateready
    • cached

    以上事件顺序触发，也可以通过调用applicationCache.update()手工触发更新。

    # 数据存储

    # Cookie

    • 内容都是值对
    • 内容大小和个数都有限制（根据浏览器不同而不同）
    • 不区分大小写
    • 有域、路径、失效时间、安全标识的限制
    • js当中的cookie会经过URL编码
    • js当中直接设置cookie不是覆盖而是追加
    • js当中没有直接删除cookie的方式，而是通过设置失效时间过期做到的
    • cookie中不要存放敏感数据，不安全。也不要搞太大，会影响性能

    # IE用户数据

    懒得看，都过时了。

    # Web存储机制

    Web Storage是一种本地存储规范。

    主要有：

    • Storage
    • sessionStorage
    • globalStorage
    • localStorage

    因为安全问题和实用性等问题，现在常用的就是seesionStorage和localStorage了

    主要方法如下：

    • clear()
    • getItem(name)
    • key(index)
    • remove(name)
    • setItem(name, value)

    不同浏览器写入磁盘的实际不同，若要强制写入磁盘可以如下操作：

    sessionStorage.begin();
sessionStorage.name = "jaybai";
sessionStorage.commit() ;

    Storage事件有如下属性：

    • domain
    • key
    • newValue
    • oldValue

    各种浏览器对本地存储的限制大小在2.5M-5M不等

    # IndexedDB

    全称是indexed Database API，是为了代替废弃的Web SQL Database API。IndexedDB整体设计都是异步的，大多数操作都会以请求的方式进行，几乎每次都需要注册onerror和onsuccess事件处理。

    因为浏览器差异所以应该这样获取DB对象实例：

    var indexedDB = window.indexedDB || window.msIndexedDB || window.mozIndexedDB || window.webkitIndexedDB；

    indexedDB具有以下特点：

    1. 键值对存储
    2. 异步
    3. 支持事物
    4. 同源策略
    5. 存储空间大
    6. 支持二进制

    主要API抽象出来的对象：

    1. 数据库：IDBDatabase 对象
    2. 对象仓库：IDBObjectStore 对象
    3. 索引： IDBIndex 对象
    4. 事务： IDBTransaction 对象
    5. 操作请求：IDBRequest 对象
    6. 指针： IDBCursor 对象
    7. 主键集合：IDBKeyRange 对象

    主要操作跟常规的DB和NOSQL差不多的
