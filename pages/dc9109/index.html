    第13章 事件

    S和HTML之间的交互是通过事件实现的，其实就是传统的观察者模式。

    # 事件流

    事件冒泡

    IE的事件流叫做事件冒泡流（event bubbling），即事件开始时由最具体的元素接受，然后逐级向上传播。现在几乎所有主流浏览器都已经支持了冒泡流。

    事件捕获

    Netscape Communicator团队提出的另外一种事件流叫做事件捕获（event capturing）。思路跟冒泡正好相反从最顶层的抽象元素开始往具体元素传递。大部分现代浏览器也实现的时间的捕获，不过用冒泡更稳妥一点。

    DOM事件流

    DOM2规定的时间流包括3个阶段：（几乎跟Android、iOS的界面事件响应机制一毛一样，部分名词叫法不同而已）

    1. 时间捕获阶段
    2. 出于目标阶段
    3. 事件冒泡阶段

    # 事件处理程序

    事件处理程序都是以“on”开头的，例如：onclick、onload

    HTML事件处理程序

    这样创建一个封装着元素属性的函数，有一个局部变量event，也就是事件对象

    <!-- 输出“click” -->
<input type="button" value="Click Me" onclick="alert(event.type)">

    在这个函数内部，this值等于时间目标元素，还可以像访问局部变量一样访问document及钙元素本身的成员，如果是表单，还能直接访问表单内的其他字段

    <!-- 输出“Click Me” -->
<input type="button" value="Click Me 1" onclick="alert(this.value)">

<!-- 输出“Click Me” -->
<input type="button" value="Click Me 2" onclick="alert(value)">

<!-- 输出“jaybai” -->
<form method="post">
	<input type="text" name"username" value="jaybai">
	<input type="button" value="Echo Username" onclick="alert(username.value)">
</form>

    HTML中指定时间处理程序缺点：

    1. 时差，需要用try-catch处理
    2. 不同浏览器中处理结果可能不一致
    3. HTML代码和JS代码耦合

    DOM0级事件处理程序

    var btn = document.getElementById("myBtn");
btn.onclick = function( ){
	alert(this.id); //"myBtn"
};

btn.onclick = null; //删除事件处理程序
    • 内部的this指向元素本身
    • 这样添加的事件处理程序在事件冒泡阶段处理

    DOM2级事件处理程序

    var btn = document.getElementById("myBtn");
btn.addEventListener("click", function( ){
	alert(this.id);
}, false);

btn.removeEventListener("click", function( ){ //无效的
	alert(this.id);
}, false);


var btn1 = document.getElementById("myBtn1");
var handler = function( ){
	alert(this.id);
};
btn1.addEventListener("click", handler, false);

btn1.removeEventListener("click", handler, false); //有效

    定义了2个方法

    • addEventListener()
    • removeEventListener()

    所有的DOM节点都包含这2个方法，接受3个参数

    1. 要处理的事件名
    2. 时间处理程序的函数
    3. 布尔值，true表示在捕获阶段，false表示在冒泡阶段

    注意2个问题：

    1. addEventListener()可以添加多个事件响应程序，按顺序执行，onclick不行
    2. addEventListener()添加的事件响应程序只能通过removeEventListener()删除。并且匿名函数无法删除，如果要删除需要把处理函数独立出来。

    IE事件处理程序

    跟DOM类似，2个方法

    • attachEvent()
    • detachEvent()

    2个参数

    1. 事件处理程序名称
    2. 时间处理程序函数

    也是在冒泡阶段处理，跟DOM0不同的是作用域，不是元素是全局this等于window

    跨浏览器的事件处理程序

    var EventUtil = {
	addHandler: function(element, type, handler){
		if(element.addEventListener){
			element.addEventListener(type, handler, false);
		}else if(element.attachEvent){
			element.attachEvent("on" + type, handler);
		}else{
			element["on" + type] = handler;
		}
	},
	removeHandler: function( ){
		if(element.removeEventListener){
			element.removeEventListener(type, handler, false);
		}else if(element.detachEvent){
			element.detachEvent("on" + type, handler);
		}else{
			element["on" + type] = null;
		}
	}
};

var btn = document.getElementById("myBtn");
var handler = function( ){
	alert("Clicked");
};

EventUtil.addHandler(btn, "click", handler);

EventUtil.removeHandler(btn, "click", handler);

    # 事件对象

    DOM上出发事件时，会产生一个evnet事件对象，会包含一些相关信息。浏览器都支持，但是可能方式不同。

    DOM中的事件对象

    所有事件的公用成员

    属性/方法 类型 读/写 说明
    bubbles Boolean 只读 时间是否冒泡
    cancelable Boolean 只读 是否可以取消事件的默认行为
    currentTarget Element 只读 其事件处理程序当前正在处理事件的那个元素
    defaultPrevented Boolean 只读 true表示已经调用了preventDefault()（DOM3事件中新增）
    detail Integer 只读 与事件相关的细节信息
    eventPhase Integer 只读 调用时间处理程序的阶段：1=捕获，2=处于目标，3=冒泡
    preventDefault() Function 只读 取消事件的默认行为，如果cancelable为true，则可以用
    stopImmediatePropagation() Function 只读 取消事件的进一步捕获或冒泡，同时阻止任何事件处理程序被调用（DOM3新增）
    stopPropagation() Function 只读 取消事件的进一步捕获或冒泡。如果bubbles为true,则可以用
    target Element 只读 事件的目标
    trusted Boolean 只读 为true表示事件是浏览器生成的。为false表示事件是由开发人员通过JavaScript创建的（DOM3中新增）
    type String 只读 被触发的事件的类型
    view AbstractView 只读 与事件关联的抽象视图。等同于发生事件window对象
    • 事件处理程序内部，this始终等于currentTatget，而target则只包含事件的实际目标。
    • 需要通过一个函数处理多个事件时，可以一同type属性switch-case去处理
    • 只有在时间处理程序执行期间，event对象才会存在

    IE中的事件对象

    • 在使用DOM0方法添加时间处理程序时，event对象作为winodw对象的一个属性存在
    • 如果是通过attachEvent()方法添加的，跟其他的浏览器类似有event对象传入

    IE中所有事件的公用成员

    属性/方法 类型 读/写 说明
    cancelBubble Boolean 读/写 默认false，设为true可以取消事件冒泡
    returnValue Boolean 读/写 默认true，设为false可以取消事件的默认行为
    srcElement Element 只读 事件的目标
    type String 只读 被触发的事件的类型

    # 事件类型

    DOM3规定了一下几类事件：

    • UI事件，当用户与页面上的元素交互时触发
    • 焦点事件，元素获取或失去焦点时触发
    • 鼠标事件，当用户通过鼠标在页面上执行操作时触发
    • 滚轮事件，当使用鼠标滚轮（或类似设备）时触发
    • 文本事件，当文档中输入文本时触发
    • 键盘事件，当用户通过键盘在页面上执行操作时触发
    • 合成事件，当为IME(输入法编辑器)输入字符时触发
    • 变动（mutation）事件，当底层DOM结构发生变化时触发
    • 变动名称事件，当元素或属性名变动时触发。已经被废弃
