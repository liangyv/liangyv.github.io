    第15章Canvas绘图

    # 基本用法

    <canvas id="jaybaiDraw" width="200" height="200">jaybai</canvas>
var a = document.getElementById("jaybaiDraw");

if(a.getContext){
	//取得2D上下文
	var context = a.getContext("2d");
	
	//取得图像的数据URI
	var imgURI = a.toDataURL("image/png");//默认就是PNG
	//显示图像
	var image = document.createElement("img");
	image.src = imgURI;
	document.body.appendChild(image);
}

    # 2D上下文

    填充和描边

    var a = document.getElementById("jaybaiDraw");

if(a.getContext){
	var context = a.getContext("2d");
	
	//这2个对象默认值是#000000，可以使用CSS中指定的任何格式，包括颜色、16进制妈、rgb、rgba、hsl、hsla
	context.strokeStyle = "red"; //描边
	context.fillStyleStyle = "#0000ff"; //填充
}

    绘制矩形

    • fillRect(x, y, width, height)
    • strokeRect(x, y, width, height)
    • clearRect(x, y, width, height)
    var a = document.getElementById("jaybaiDraw");

if(a.getContext){
	var context = a.getContext("2d");

	context.strokeStyle = "#ff0000";
	context.strokeRect(10, 10, 50, 50);
	
	context.fillStyle = "rgba(0, 0, 255, 0.5)";
	context.fillRect(30, 30, 50, 50);
	
	context.clearRect(40, 40, 10 ,10);
}
    • lineWidth属性控制描边宽度，可以是任意整数。
    • lineCap属性控制线条末端的形状是平头、圆头还是方头（butt、round、square）
    • lineJoin属性控制先挑相交的方式是圆交、斜交还是斜接（round、bevel、miter）

    绘制路径

    首先必须调用beginPath()方法，然后再实际调用下列方法：

    • arc(x, y, radius, startAngle, endAngle, counterclockwise)：以（x,y）为圆心绘制一条弧线，半径为radius，起始和结束（用弧度表示）分别是startAngle、endAngle。counterclockwise为false表示逆时针计算startAngle、endAngle。
    • arcTo(x1, y1, x2, y2, radius)：从上一点开始绘制一条弧线，到(x2,y2)为止，并且以给定的半径radius穿过（x1,y1）
    • bezierCurveTo(c1x, c1y, c2x, c2y, x, y)：从上一个点开始绘制一条曲线，到(x,y)为止，并且以（c1x,c1y）和（c2x,c2y）为控制点
    • lineTo(x, y)：从上一点开始绘制一条直线到（x,y）为止
    • moveTo(x, y)：讲绘图游标移动到(x,y)，不画线
    • quadraticCurveTo(cx, cy, x, y)：从上一点开始绘制一条二次曲线，到(x,y)为止，并且以()
    • rece(x, y , width, height)

    创建路径后有几种选择：

    1. closePath()：绘制一条连接到路径起点的线条
    2. fill()：用fillStyle填充它
    3. stroke()：用strokeStyle填充它
    4. clip()：在路径是那个创建一个剪切区域
    //路径被关闭前确认画布上的某一个点是否位于路径上
if(context.isPonitInPath(100, 100)){
	alert(Point(100,100) is in the path.);
}

    绘制文本

    • fillText(text, x, y, maxPixelWidth)
    • strokeText(text, x, y, maxPixelWidth)

    这2个方法都以一下3个属性为基础：

    1. font：样式、大小及字体，例如“10px Arial”
    2. textAlign：对齐方式，值有start、end、left、right、center
    3. textBaseline：本文基线，值有top、hanging、middle、alphabetic、ideographic、bottom

    变换

    • rotate(angle)：围绕原点旋转图像angle弧度
    • scale(a, b)：缩放图像，在xy方向上分别乘以a、b。默认ab都是1.0
    • translate(x, y)：将坐标原点移动到(x,y)
    • transform(m11, m12, m21, m22, dx, dy)：直接修改改变矩阵，直接乘以矩阵
    • setTransform(m11, m12, m21, m22, dx, dy)：先重置矩阵，在调用transform

    矩阵是这样的的

    m11 m12 dx
    m21 m22 dy
    0 0 1

    绘制图像

    • drawImage()

    有3种调用方式

    1. drawImage(image, x, y)
    2. drawImage(image, x, y, width, height)
    3. drawImage(image, src_x, y, src_width, src_height, src_target_x, target_y, target_width, target_height)

    阴影

    • shadowColor，默认黑色
    • shadowOffsetX，默认0
    • shadowOffsetY，默认0
    • shadowBlur，默认0不模糊

    渐变

    CancasGradient实例表示渐变。

    • createLinearGradient(start_x, start_y, end_x, end_y)：创建线性渐变
    • addColorStop(pos, color)：指定色标位置和颜色，0=开始，1=结束

    把fillStyle或strokeStyle设置为这个对象，就可以渐变的填充或者描边了。

    一般为了渐变更充分，设置的位置要等于你需要渐变的区域或者元素的坐标大小

    模式

    模式就是重复的图像。

    • createPattern(image, patternStr)：patternStr跟CSS的background-repeat属性值相同，repeat、repeat-x、repeat-y、no-repeat

    使用图像数据

    • getImageData(x, y, width, heigth)

    取得一个ImageData对象，都有width,heigth和data对象，data属性是一个数组，保存着图像每一个像素的数据。每一个像素用4个元素保存，分别表示红、绿、蓝和透明度。

    合成

    • globalAlpha，全局透明度。介于0-1之间包括0、1，绘制的透明度，默认0
    • globalCompositionOperation，后绘制的图像怎么与先绘制的图像结合。

    globalCompositionOperation的取值可能是：

    • source-over：默认值，后绘的再上方
    • source-in：后绘制的与先绘制的重叠部分可见，两者其他完全透明
    • source-out：后绘制的与先绘制的不重叠部分可见，先绘制的完全透明
    • source-atop：后绘制的与先绘制的重叠部分可见，先绘制的图形不受影响
    • destination-over：后绘的在下方
    • destination-in：后绘的在下方，其他同理
    • destination-out：后绘的在下方，其他同理
    • destination-atop：后绘的在下方，其他同理
    • lighter：后绘制的与先绘制的重叠部分的值相加，该部分高亮
    • copy：后绘制的代替先绘制的重叠部分
    • xor：后绘制的部分和先绘制的部分进行异或操作

    # WebGL

    WebGL是针对Canvas的3D上下文。

    类型化数组

    为了解决JavaScript数值精度问题，WebGL引入了一个概念叫做类型化数组（typed arrays）。核心是一个名为ArrayBuffer的类型。只分配一定的字节数量，不指定类型。

    var buffer = new ArrayBuffer(20);
var bytes = buffer.byteLength;

    使用ArrayBuffer的一种特别的方式就是用它来创建数组缓冲器视图。最常见的就是DataView，通过它可以选择ArrayBuffer中的一小段字节。实例化后，DataView对象有2个属性：

    1. byteOffset
    2. byteLength
    数据类型 getter setter
    有符号8位整数 getInt8(byteOffset) setInt8(byteOffset, value)
    无符号8位整数 getUint8(byteOffset) setUint8(byteOffset, value)
    有符号16位整数 getInt16(byteOffset, littleEndian) setInt16(byteOffset, value, littleEndian)
    无符号16位整数 getUint16(byteOffset, littleEndian) setUint16(byteOffset, value, littleEndian)
    有符号32位整数 getInt32(byteOffset, littleEndian) setInt32(byteOffset, value, littleEndian)
    无符号32位整数 getUint32(byteOffset, littleEndian) setUint32(byteOffset, value, littleEndian)
    32位浮点数 getFloat32(byteOffset, littleEndian) setFloat32(byteOffset, value, littleEndian)
    64位浮点数 getFloat64(byteOffset, littleEndian) setFloat64(byteOffset, value, littleEndian)

    无符号8位整数要用1B，32位浮点数则要用4B，需要自己管理这些细节。

    类型化数组继承自DataView

    • Int8Array
    • Uint8Array
    • Int16Array
    • Uint16Array
    • Int32Array
    • Uint32Array
    • Float32Array
    • Float64Array

    每个视图构造函数都有一个名为BYTES_PER_ELEMENT的属性，表示类型化数组的元素需要多少个字节

    WebGL上下文

    WebGL的名字叫做“experimental-webgl”规范后会变成“webgl”

    getContent()的第二个参数是一个对象，可以包含如下属性

    • alpha，true表示创建一个Alpha通道，默认是true
    • depth，true表示使用16位色深，默认true
    • stencil，true表示使用8位模板缓冲区，默认false
    • antialias，true表示使用默认机制抗锯齿，默认值为true
    • premultipliedAlpha，true表示绘图缓冲区有预乘Alpha值，默认为true
    • preserveDrawingBuffer，true表示绘图完成保存绘图缓冲区，默认false。开启会影响性能，有必要才开
    1. 常量以GL_前缀开头
    2. 方法名的后缀会包含参数个数和类型（f=浮点，i=整数）
    3. 操作WebGL之前用clearColor()指定颜色清除canvas
    4. 可以用veiwport()调整视口
    5. 可以用createBuffer()然后bindBuffer()绑定到WebGL上下文，然后就可以用数据填充缓冲区了
    6. getError()可以拿到错误
    7. 有定点着色器和片段着色器
    8. CLSL是一种类C语言，专门用于编写OpenGL着色器
    9. createShader()方法传入要创建的着色器类型，用compileShader()编译
    10. 通过getUniformLocation()获得Uniform变量在内存中的位置
    11. getShaderParameter()取得着色器的编译状态
    12. drawArrays()和drawElements()绘图，前者用于数据缓冲区，后者用于元素缓冲区
    13. createTexture()创建纹理，必须在load事件以后才能设置纹理
    14. readPixels()取得像素值
