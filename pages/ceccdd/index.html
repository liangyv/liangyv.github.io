    第6章 面向对象的程序设计

    ES中的对象跟java、c/c++中的不太一样，可以理解成散列表（一堆key对应了一堆数据/函数），弱类型的也不存在什么预先定义的问题，随时都可以增删改属性和方法，简直是奔放。。。

    ES5中定义了内部才能访问的属性,类似[[Enumable]]这样，但是主要为实现js引擎用的，js中直接访问不了

    ES中有2种属性：

    1. 数据属性
    2. 访问器属性

    数据属性有4个特性:

    1. [[Configurable]]：能否删、改属性，能否修改特性，能否改为访问器属性。大部分默认true
    2. [[Enumerable]]：能否通过for-in循环。大部分默认true
    3. [[Wirtable]]：能否修改属性的值。大部分默认true
    4. [[Value]]：这个属性的数据值。默认值是undefined

    ES5提供了Object.defineProperty(属性所在对象, 属性名字, 描述符对象)方法修改属性值。

    var o = {};
//o对象的name属性置为了不可写状态
Object.defineProperty(o, "name", {
	wirteable: false,
	value: "jaybai"
});

alert(o.name); //jaybai
o.name = "baijie";
alert(o.name); //jaybai

    **不可配置属性(configurable)**一旦被设成false后就再也没法通过Object.defineProperty()方法去修改这个对象的属性了

    访问器属性不包含数据值，有如下4个特性：

    1. [[Configurable]]：能否删、改属性，能否修改特性，能否改为访问器属性。大部分默认true
    2. [[Enumerable]]：能否通过for-in循环。大部分默认true
    3. [[Get]]：在读取属性时调用。默认值是undefined
    4. [[Set]]：在设置属性时调用。默认值是undefined

    访问器属性不能直接定义，必须用Object.defineProperty()方法定义

    //name前面的下划线用于表示只能通过对象方法访问的属性,但是这里其实只是一个约定俗成，并没有什么卵用，想访问还是可以直接访问到
var o = {
	_name: "jaybai",
	age: 18
};

//setter和getter可以单独指定，不指定就不能用，忽略或者返回undefined
Object.defineProperty(o, "name", {
	get: function( ){
		return this._name;
	},
	set: function(x){
		this._name = x;
		this.age++;
	}
});

o.name = "baijie";
alert(o.name); //baijie
alert(o.age); //19
//定义多个属性
var o = {};
Object.defineProperties(o, {
	_name:{
		value: "jaybai"
	},
	age:{
		value: 18
	},
	name:{
		get: function( ){
			return this._name;
		},
		set: function(x){
			this._name = x;
			this.age++;
		}
	}
});

//读取属性的特征
var a = Object.getOwnPropertyDescriptor(o, "_name");
alert(a.value); //jaybai
alert(a.configurable); //false

    # 创建对象

    工厂模式

    ES没得类，所以只能用函数来骚操作实现，而且还无法限制这些对象属性的类型，蛋疼。

    function createCoder(name, age, language){
	var o = new Object();
	o.name = name;
	o.age = age;
	o.language = language;
	o.coding = function( ){
		alert(this.name + " coding " + this.language + " code.");
	}
	return o;
}

var coder1 = createCoder("jaybai", 18, "JavaScript");
var coder2 = createCoder("baijie", 18, "Java");
var coder3 = createCoder("bulargy", 18, "C++");

    构造函数模式

    //构造器函数首字母大写，约定俗成，也没卵限制
function Coder(name, age, language){
	this.name = name;
	this.age = age;
	this.language = language;
	this.coding = function( ){
		alert(this.name + " coding " + this.language + " code.");
	}
}

//这个操作就跟java、c/c++长得差不多的,new对象出来
var coder1 = new Coder("jaybai", 18, "JavaScript");
var coder2 = new Coder("baijie", 18, "Java");
var coder3 = new Coder("bulargy", 18, "C++");

//构造函数属性
alert(coder1.constructor == Coder); //true

alert(code1 instanceof Object); //true
alert(code1 instanceof Coder); //true

    跟工厂模式的区别：

    1. 没有显示的创建对象
    2. 属性和方法直接付给了this
    3. 没有return

    new操作符调用构建函数经历4步：

    1. 创建一个新对象
    2. 将构造函数的作用域付给新对象（this指向这个对象）
    3. 执行构造函数的代码
    4. 返回新对象

    构造函数也是函数，其实没卵区别，只是看你用不用new操作符而已。所以又可以开始骚操作了。

    function Coder(name, age, language){
	this.name = name;
	this.age = age;
	this.language = language;
	this.coding = function( ){
		alert(this.name + " coding " + this.language + " code.");
	}
}

//当做构造函数
var coder = new Coder("jaybai", 18, "JavaScript");
coder.coding(); //jaybai coding JavaSrcipt code.

//当做普通函数
Coder("baijie", 18, "Java"); //添加到window
window.coding(); //baijie coding Java code.

//在另一个对象的作用于中调用
var o = new Object();
Coder.call(o, "bulargy", 18, "C++")
o.coding(); //bulargy coding C++ code.

    构造函数的方式会导致重复的同名方法创建

	this.name = name;
	this.age = age;
	this.language = language;
	this.coding = function( ){
		alert(this.name + " coding " + this.language + " code.");
	}
}

var coder1 = new Coder("jaybai", 18, "JavaScript");
var coder2 = new Coder("baijie", 18, "Java");

alert(coder1.coding == coder2.coding); //false

    可以把函数定义移动到构造函数外部

    function Coder(name, age, language){
	this.name = name;
	this.age = age;
	this.language = language;
	this.coding = coding;
}

function coding( ){
	alert(this.name + " coding " + this.language + " code.");
}

var coder1 = new Coder("jaybai", 18, "JavaScript");

alert(coder1.coding == coder2.coding); //true

    以上修改还是有问题，一是全局函数只能给某个对象用；二是如果很多方法，那就毫无封装性可言。

    原型模式

    对象实例可以共享原型对象的属性和方法。

    function Coder( ){
}

Coder.prototype.name = "jaybai";
Coder.prototype.age = 18;
Coder.prototype.language = "JavaScript";
Coder.prototype.coding = function( ){
    alert(this.name + " coding " + this.language + " code.");
};

var coder1 = new Coder();
var coder2 = new Coder();

coder1.coding(); //jaybai coding JavaSrcipt code.
coder2.coding(); //jaybai coding JavaSrcipt code.

    ECMA-265第5版有一个指针叫做[[Prototype]],用来指向对象实例的构造函数的原型，不能直接访问。但是firefox、safari、chrome都支持一个属性__proto__。 例如： coder1的[[Prototype]] == coder2.[[Prototype]] == Coder.prototype

    构造函数原型有一个constructor方法又指向构造函数本身 例如: Coder.prototype.constructor == Coder

    function Coder( ){
}

Coder.prototype.name = "jaybai";
Coder.prototype.age = 18;
Coder.prototype.language = "JavaScript";
Coder.prototype.coding = function( ){
    alert(this.name + " coding " + this.language + " code.");
};

var coder1 = new Coder();
var coder2 = new Coder();

alert(Coder.prototype.isPrototypeOf(coder1)); //true
alert(Coder.prototype.isPrototypeOf(coder2)); //true

//ES5新增的Object.getPrototypeOf()返回对象实例的[[Prototype]]
alert(Object.getPrototypeOf(coder1) == Coder.prototype); //true
alert(Object.getPrototypeOf(coder1).name); //jaybai

    直接修改原型可以立刻在实例中体现出来，但是如果重写了整个原型，会切断构造函数创建的实例指向原型的指针。所以实例中的[[Prototype]]指针仅仅指向原型而不指向构造函数。

    function Coder( ){
}

var c1 = new Coder();
Coder.prototype.sayHi = function( ){
    alert("Hi");
};
c1.sayHi();//Hi

Coder.prototype = {

    language : "JavaScript",
    coding : function( ){
        alert(this.name + " coding " + this.language + " code.");
    }
};
var allKeys = Object.getOwnPropertyNames(c1.__proto__);
alert(allKeys); //constructor,sayHi
c1.coding();//报错了，因为重写了Coder的prototype，c1的[[Prototype]]还是指向原来的构造函数构造的原型，该原型中没有coding方法

    不仅仅自定义对象，原生引用类型（Object、Array、String等）都在其狗在函数的原型上定义了方法。所以又可以骚操作了。虽然可以骚操作，但是强烈不建议这么搞，非常容易翻车。

    String.prototype.hehe = function( ){
    alert("hehe");
};
var a = "aaaaa";
a.hehe(); //hehe

function Coder(name, age, language){

    s.toXString = function( ){

    }
    return s;
}

var x = new XString("aaa");

    稳妥构造函数模式

    到克拉斯.克罗克福德（Douglas Crockford）发明了JavaScript中的稳妥对象（durable object）这个概念。指的是没有公共属性，而且其方法也不引用this的对象。适合在安全环境下使用

    function Coder(name, age, language){
	var o = new Object();
	var name = name;
	var age = age;
	var language = language;
    
	o.coding = function( ){
		alert(name + " coding " + language + " code.");
	}
	return o;
}

//只能这样访问内部属性
var coder1 = Coder("jaybai", 18, "JavaScript");
coder1.coding(); //jaybai coding JavaScript code.
//直接访问内部变量访问不到
alert(coder1._name); //undefined

//new出来的也访问不到
var coder2 = new Coder("jaybai", 18, "JavaScript");
coder2.coding(); //jaybai coding JavaScript code.
alert(coder2._name); //undefined

    # 继承

    OO继承：

    • 接口继承
    • 实现继承

    ES方法没有签名，所以没有办法接口继承，只有靠原型链来实现实现继承。

    原型链

    function SuperType( ){
    this.property = true;//SuperType的实例方法
}

SuperType.prototype.getSuperValue() = function( ){
    return this.property; //SuperType的原型属性
};

function SubType( ){
    this.subproperty = false; //SubType的实例方法
}

//SubType继承了SuperType
SubType.prototype = new SuperType(); //注意！！！千万不能使用字面量定义原型

//新增or重写超类型的方法，一定要放在替换原型的语句后面（下面这句一定要放在上面那句后面）
SubType.prototype.getSubValue = function( ){
    return this.subproperty; //SubType的原型属性
};

var instacne = new SubType();
alert(instance.getSuperValue()); //true

//只要实例和原型链出想过的构造函数都可以用instanceof测试到
alert(instance instanceof Object); //true
alert(instance instanceof SuperType); //true
alert(instance instanceof SubType); //true

//只要是原型链中出现过的原型，都可以说是该原型链所派生的实例的原型
alert(Object.prototype.isPrototypeOf(instacne)); //true
alert(Object.SuperType.isPrototypeOf(instacne)); //true
alert(Object.SubType.isPrototypeOf(instacne)); //true

    原型链的问题：

    1. 会共享引用类型的值。（跟原型属性会被实例共享一个道理，因为超类型的实例属性就变成了子类型的原型属性）
    2. 创建子类型实例时，不能像超类型的构造函数传递参数。（准确的说是不能要传也行，但是会影响对象实例）
    function SuperType( ){
    this.names = ["aa", "bb", "cc"];
}

function SubType( ){
}

SubType.prototype = new SuperType();

var a = new SubType();
a.names.push("dd");
alert(a.names); //aa,bb,cc,dd

var b = new SubType();
alert(b.names); //aa,bb,cc,dd —— 被a的操作影响了

    借用构造函数(construtor stealing)

    又叫做伪造对象或者经典继承，即在子类型构造函数的内部调动超类型的构造函数。

    function SuperType(nickname){
    this.names = ["aa", "bb", "cc"];
    this.nickname = nickname;
}

function SubType( ){
    //继承了SuperType，还能传参数
    SuperType.call(this, "jaybai"); //apply()也可以
    this.age = 18;
}

var a = new SubType();
a.names.push("dd");
alert(a.names); //aa,bb,cc,dd

//这里执行构造函数SubType()的时候执行了SuperType()，把names又初始化成了aa,bb,cc
var b = new SubType(); 
alert(b.names); //aa,bb,cc
alert(a.nickname); //jaybai
alert(a.age); //18

    借用构造函数也无法避免构造函数模式的问题——方法都在狗仔函数中定义，无法复用。而且超类型原型中定义的方法，对子类型而言也是不可见的，导致搜又类型都只能使用构造函数模式。

    组合继承（combination inheritance）

    也叫伪经典继承，试讲原型链和借用构造函数组合到一块。思路是用原型链实现对原型属性和方法的继承，铜鼓哦借用构造函数实现对实例属性的继承。（最常用的继承方式）

    function SuperType(nickname){
    this.names = ["aa", "bb", "cc"];
    this.nickname = nickname;
}

SuperType.prototype.sayNickname = function( ){
    alert(this.nickname);
};

function SubType(nickname, age){
    //继承属性
    SuperType.call(this, nickname);
    this.age = age;
}

//继承方法
SubType.prototype = new SuperType();

SubType.prototype.sayAge = function( ){
    alert(this.age);
};

var a = new SubType("jaybai", 18);
a.names.push("dd");
alert(a.names); //aa,bb,cc,dd
a.sayNickname(); //jaybai
a.sayAge(); //18

var b = new SubType("baijie", 19);
alert(b.names); //aa,bb,cc
b.sayNickname(); //baijie
b.sayAge(); //19

    原型式继承

    道格拉斯.克罗克福德在2006年写了一篇文章，题为Prototypal Inheritance in JavaScript（JavaScript中的原型式继承）。 借助远行可以基于已有的对象创建新对象，同时还不必因此创建自定义类型

    function object(o){
    function F( ){}
    F.prototype = o;
    return new F();
}

    本质上object()对传入其中的对象执行了一次浅复制

    function object(o){
    function F( ){}
    F.prototype = o;
    return new F();
}

var coder = {
    name : "jaybai",
    language : ["JavaScript", "Java", "C++"]
};

var a = object(coder);
a.name = "baijie";
a.language.push("Golang");
alert(a.language); //JavaScript,Java,C++,Golang

var b = object(coder);
b.name = "bulargy";
b.language.push("Kotlin");
alert(b.language); //JavaScript,Java,C++,Golang,Kotlin

alert(coder.language); //JavaScript,Java,C++,Golang,Kotlin

    ES5中新增Object.create()方法规范了原型继承

    var coder = {
    name : "jaybai",
    language : ["JavaScript", "Java", "C++"]
};

var a = Object.create(coder);
a.name = "baijie";
a.language.push("Golang");
alert(a.language); //JavaScript,Java,C++,Golang

var b = Object.create(coder, {
    name:{
        value : "bulargy"
    }
});
b.language.push("Kotlin");
alert(b.name); //bulargy
alert(b.language); //JavaScript,Java,C++,Golang,Kotlin

    注意！！！原型式继承跟原型模式一样，引用类型始终都会共享

    寄生式(parasitic)继承

    也是克洛福德推广的

    function createObject(o){
    var clone = Object.cerate(o); //通过调用函数创建一个新对象
    clone.sayHi = function( ){ //增强这个对象
        alert("hi");
    };
    return clone; //返回这个对象
}

var coder = {
    name : "jaybai",
    language : ["JavaScript", "Java", "C++"]
};

var a = createObject(coder);
a.sayHi(); //hi

    注意！！！寄生式继承***不能做到函数复用***而效率降低，跟构造函数模式类似。

    寄生组合式继承

    //回顾一下组合继承最大的问题就是无论什么情况下，都会调用两次超类型的构造函数

function SuperType(nickname){
    this.names = ["aa", "bb", "cc"];
    this.nickname = nickname;
}

SuperType.prototype.sayNickname = function( ){
    alert(this.nickname);
};

function SubType(nickname, age){
    SuperType.call(this, nickname); //第二次调用SuperType()
    this.age = age;
}

SubType.prototype = new SuperType(); //第一次调用SuperType()

SubType.prototype.sayAge = function( ){
    alert(this.age);
};

var instance = new SubType("Nicholas", 29);

    所谓寄生这式继承，即通过借用构造函数来继承属性，通过原型链的混成形式来继承方法。

    function inheritPrototype(subType, superType){
	var pt = Object.create(superType.prototype); //创建对象
	pt.constructor = subType; //增强对象
	subType.prototype = pt; //指定对象
} 

function SuperType(nickname){
    this.names = ["aa", "bb", "cc"];
    this.nickname = nickname;
}

SuperType.prototype.sayNickname = function( ){
    alert(this.nickname);
};

function SubType(nickname, age){
    SuperType.call(this, nickname);  //只一次调用SuperType()
    this.age = age;
}

//没有了SubType.prototype = new SuperType();语句，函数内部直接是吧增强的SuperType.prototype复给了SubType.prototype。因此避免了SubType.prototype上创建不必要的、多余的属性。
inheritPrototype(SubType, SuperType); 

SubType.prototype.sayAge = function( ){
    alert(this.age);
};

    开发人员普遍认为寄生组合式继承是引用类型最理想的继承方式。
